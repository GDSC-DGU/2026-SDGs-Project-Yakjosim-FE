# 약물·음식·영양제 상호작용 안내 서비스 기능 명세서

## 1. 서비스 개요

### 1-1. 서비스명

약 조심

### 1-2. 서비스 목표 및 핵심 가치

사용자가 **약 이름, 처방전 사진, 영양제명, 음식명**을 입력하면 해당 조합의 위험 여부를 분석하고, 일반인이 이해할 수 있는 언어로 복용 주의사항을 안내하는 서비스

약학정보원은 의약품 검색, DUR, 약물-약물 상호작용, 약물-음식 상호작용 정보를 제공하고 있으며, 식품의약품안전처/공공데이터포털의 DUR 품목정보 API는 병용금기, 특정연령대 금기, 임부금기, 용량주의, 투여기간주의, 노인주의, 효능군 중복주의 등의 정보를 제공합니다. 따라서 본 서비스는 해당 공신력 있는 데이터를 사용자 친화적 UI와 AI 설명 계층으로 변환하는 구조가 적합합니다. ([약학정보원](https://www.health.kr/?utm_source=chatgpt.com))

### 1-3. 핵심 가치

| 구분 | 설명 |
| --- | --- |
| 접근성 | 성분명을 몰라도 제품명 또는 처방전 사진으로 검색 가능 |
| 안전성 | 약물-약물, 약물-음식, 약물-영양제 위험 조합 사전 안내 |
| 이해성 | “왜 위험한지”, “얼마나 간격을 둬야 하는지”를 쉬운 문장으로 설명 |
| 신뢰성 | AI가 임의로 판단하지 않고 공공·전문 데이터 기반으로 답변 |
| 실용성 | “복용 금지”, “시간 차 복용”, “약사 상담 필요” 등 행동 가이드 제공 |

### 1-4. 주요 타겟 고객

| 타겟 | 필요성 |
| --- | --- |
| 다약제 복용 고령층 및 보호자 | 처방약이 많아 병용금기·중복복용 위험이 큼 |
| 영양제를 함께 복용하는 2030~5060 사용자 | 건강기능식품과 약의 상호작용을 확인하기 어려움 |
| 만성질환자 | 고혈압약, 당뇨약, 갑상선약, 항응고제 등 장기 복용 약물 관리 필요 |
| 약국 방문이 어렵거나 의료 접근성이 낮은 사용자 | 기본 복약 주의사항을 빠르게 확인할 수 있음 |
| 약 이름은 알지만 성분명을 모르는 일반 사용자 | 제품명 기반 검색 필요 |

### 1-5. 서비스 범위

이 서비스는 **의료 진단·처방 서비스가 아니라 복약 안전 정보 안내 서비스**입니다.

서비스 내 모든 결과에는 다음 원칙을 적용합니다.

- 약 복용 중단, 변경, 대체를 직접 지시하지 않음
- 고위험 조합은 “의사,약사 상담 필요”로 안내
- AI 답변은 반드시 데이터 근거와 함께 제공
- 데이터에 없는 조합은 “안전함”이 아니라 “확인된 정보 없음”으로 표시

## 2. 사용자 흐름 User Flow

## 2-1. 전체 사용자 흐름

```
서비스 접속
→ 약 이름 검색 또는 처방전 사진 업로드
→ 제품명/성분명 자동 매핑
→ 함께 먹는 약·영양제·음식 추가 입력
→ 상호작용 분석
→ 위험도 결과 확인
→ AI 쉬운 설명 확인
→ 복용 간격/상담 필요 여부 확인
→ 결과 저장 또는 공유
```

## 6-2. MVP 권장 사용자 시나리오

### 시나리오 1. 약 이름으로 위험 조합 확인

```
사용자: “고혈압약이랑 자몽주스 같이 먹어도 되나?”
→ 약 이름 검색
→ 자몽주스 선택
→ 결과: 주의/상담 권장
→ 설명: 약 성분 농도가 높아질 수 있음
→ 행동: 복용 중 자몽 섭취 전 약사 상담 권장
```

### 시나리오 2. 처방전 사진으로 한번에 분석

```
사용자: 처방전 사진 업로드
→ OCR로 약품명 4개 추출
→ 사용자가 인식 결과 확인
→ 시스템이 성분명으로 변환
→ 병용금기/중복성분/음식 주의 분석
→ 결과 요약 제공
```

### 시나리오 3. 약과 영양제 조합 확인

```
사용자: 갑상선약 + 칼슘 영양제 입력
→ 흡수 방해 가능성 분석
→ 결과: 시간 간격 필요
→ 설명: 같은 시간 복용 시 약 흡수가 저하될 수 있음
→ 행동: 복용 간격 확인 및 전문가 상담 권장
```

## 7-1. “위험 조합”의 기준을 명확히 해야 함

단순히 “나쁨/좋음”으로 표시하면 의료정보 서비스로서 위험합니다. 다음처럼 분류하는 것이 좋습니다.

| 등급 | 의미 | 사용자 문구 |
| --- | --- | --- |
| 금기 | 함께 복용하면 안 되는 것으로 관리되는 조합 | “함께 복용 전 반드시 의사·약사와 상담하세요.” |
| 고위험 주의 | 약효 증가, 부작용 증가 가능성 | “복용 중이라면 전문가 확인이 필요합니다.” |
| 중간 주의 | 흡수 저하, 효과 감소 가능성 | “시간 간격을 두는 것이 권장될 수 있습니다.” |
| 낮은 주의 | 일반적 주의 필요 | “복용 습관을 확인하세요.” |
| 정보 없음 | 확인된 상호작용 데이터 없음 | “안전하다는 의미는 아니며, 정보가 확인되지 않았습니다.” |

## 7-2. AI는 판단자가 아니라 “설명 변환기”로 두는 것이 안전함

AI가 직접 위험 여부를 판단하면 오류 가능성이 큽니다. 구조는 다음이 적절합니다.

```
공식 데이터 조회
→ 룰 기반 위험도 산정
→ AI가 쉬운 설명으로 변환
→ 근거와 함께 결과 제공
```

즉, AI의 역할은 다음으로 제한하는 것이 좋습니다.

- 어려운 의학 용어를 쉬운 문장으로 바꾸기
- 복용 간격, 상담 필요 여부를 사용자 친화적으로 설명하기
- 결과 요약 생성
- 약사에게 보여줄 질문 문장 생성

---

## 7-3. OCR은 자동 분석보다 “사용자 확인”이 더 중요함

처방전 OCR은 편리하지만 약 이름을 잘못 인식하면 분석 결과 전체가 틀어질 수 있습니다.

따라서 OCR 결과는 반드시 다음 단계를 거쳐야 합니다.

```
OCR 추출
→ 약품명 후보 표시
→ 사용자 직접 확인
→ 성분명 매핑
→ 상호작용 분석
```
