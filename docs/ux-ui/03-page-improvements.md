# Page Improvements

## Onboarding

대상 파일: `src/pages/OnboardingPage.tsx`

개선 포인트:

- 현재 카드형 소개 3개와 동의 체크박스 구조는 단순하지만, 시작 전 신뢰 형성이 약하다.
- "왜 필요한 앱인지", "누가 쓰면 좋은지", "개인정보/의료 한계"가 더 명확히 구분되면 좋다.
- 체크박스 라벨이 길어서 모바일에서 읽기 부담이 있다.

권장 작업:

- 체크 영역 전체를 탭 가능하게 확장
- 핵심 가치 3개를 아이콘 설명보다 더 짧고 굵게 재구성
- 시작 버튼 위에 "데모 데이터 기반" 여부 또는 안내 문구 정리

## Home

대상 파일: `src/pages/HomePage.tsx`

개선 포인트:

- 상단 검색 입력은 실입력처럼 보이지만 실제로는 이동 트리거다.
- 액션 카드 4개는 기능 진입에는 좋지만 중요도 차이가 드러나지 않는다.
- 최근 기록 리스트는 결과 상세가 아니라 `/results`로만 이동해 문맥이 약하다.

권장 작업:

- 검색창을 "검색 진입 카드"처럼 시각적으로 명확히 바꾸거나 실제 인라인 검색으로 전환
- 가장 자주 쓰는 액션 1개를 primary, 나머지는 secondary 구조로 재배치
- 최근 기록 카드에 분석 대상 수와 위험도 외에 "분석 종류" 표시

## Search

대상 파일: `src/pages/SearchPage.tsx`, `src/components/common/SearchInput.tsx`, `src/components/common/MedicineCard.tsx`

개선 포인트:

- 선택된 약과 검색 결과의 위계가 약하다.
- 결과 없음 상태가 없다.
- 하단 고정 CTA와 하단 네비 간 거리 규칙이 명확하지 않다.

권장 작업:

- 검색 결과 영역 제목 추가: 예. "검색 결과", "선택한 약"
- 결과 없음 상태 추가
- 선택한 약이 많아질 때 요약 바 또는 접기 UI 고려
- CTA를 공통 하단 액션 바로 교체

## OCR

대상 파일: `src/pages/OcrPage.tsx`

개선 포인트:

- 업로드 드롭존은 명확하지만 모바일 카메라 촬영 중심 UX는 아직 약하다.
- OCR 결과가 삭제만 가능하고 수정은 안 된다.
- 신뢰도 뱃지는 있으나 사용자가 무엇을 해야 하는지 단계별 설명이 부족하다.

권장 작업:

- "촬영 팁" 추가: 빛 반사, 초점, 처방전 전체 포함 등
- 저신뢰 결과 편집 UX 제공
- 결과 확정 전 "자동 인식 결과를 확인하세요" 가이드 문구 추가
- 업로드 오류 유형을 파일 형식/인식 실패/네트워크 실패로 분리

## Combination

대상 파일: `src/pages/CombinationPage.tsx`

개선 포인트:

- 약, 음식, 영양제 선택 영역이 한 화면에 다 있어 길어진다.
- 칩 수가 많아지면 스캔성이 떨어진다.
- 분석 가능 조건이 하단 작은 문장으로만 안내된다.

권장 작업:

- 섹션 접기 또는 탭 전환 도입 검토
- 선택된 음식/영양제 요약을 상단 고정
- 분석 가능 조건을 진행형 안내로 변경
- 위험도에 영향 큰 카테고리부터 노출 순서 재검토

## Results

대상 파일: `src/pages/ResultsPage.tsx`

개선 포인트:

- 현재 요약 카드, 필터 탭, 결과 카드 구조는 합리적이다.
- 다만 가장 중요한 next action이 흐릿하다.
- "고위험"과 "낮은 주의"가 같은 리스트 밀도로 보여 우선순위 인지가 약하다.

권장 작업:

- `critical/high` 결과를 상단 우선 노출하는 요약 블록 추가
- 결과 카드에 "무엇을 해야 하는지" 한 줄을 더 직접적으로 제시
- 저장/공유보다 상담 또는 상세 확인을 더 강조할지 제품 방향 결정
- 탭 필터 개수 배지 추가

## Detail

대상 파일: `src/pages/DetailPage.tsx`

개선 포인트:

- 정보 구조는 좋지만 장문 텍스트 중심이라 스캔성이 떨어진다.
- 근거 정보 아코디언은 유용하지만 전문가용 정보와 일반 사용자용 정보가 섞여 있다.

권장 작업:

- 상단에 "위험 이유", "해야 할 행동", "상담 필요 여부"를 요약 카드로 재구성
- 설명 본문은 bullet 또는 step 형태로 분해
- 시간 간격 안내는 더 강한 시각 강조 필요
- 출처 정보는 일반 사용자용/전문가용으로 분리

## Share

대상 파일: `src/pages/SharePage.tsx`

개선 포인트:

- 공유 텍스트 생성은 되어 있으나 UI 피드백은 `alert()`에 의존한다.
- 이미지 저장은 미구현인데 버튼이 전면 노출된다.
- 공유 미리보기 카드가 결과가 많을 때 너무 길어진다.

권장 작업:

- 복사 성공/실패를 토스트 또는 인라인 피드백으로 교체
- 미구현 기능은 숨기거나 "준비 중" 배지로 명확히 표시
- 결과 3개 이상은 요약형 미리보기 + 전체보기 토글 고려

## MyPage

대상 파일: `src/pages/MyPage.tsx`

개선 포인트:

- "내 복용 목록"이라는 페이지 제목과 실제 저장 데이터 구조가 완전히 일치하지 않는다.
- 저장 약, 분석 기록, 건강 정보가 모두 같은 무게로 놓여 있다.
- 건강 정보 입력이 숫자/토글 위주라 실제 의사결정 맥락이 약하다.

권장 작업:

- 저장 약을 최상단 primary 영역으로 유지할지 재검토
- 분석 기록 카드에 다시보기, 삭제 등 후속 액션 정의
- 건강 정보가 분석 결과에 어떻게 반영되는지 설명 추가

## Settings

대상 파일: `src/pages/SettingsPage.tsx`

개선 포인트:

- 설정 화면이 정보/접근성/초기화가 단순 나열이다.
- `seniorMode`는 현재 연결 미완성 상태다.

권장 작업:

- 접근성 관련 설정을 별도 섹션으로 강화
- 고령층 모드가 실제 어떤 변화인지 미리보기 제공
- 데이터 초기화 버튼은 더 낮은 시각 우선순위로 배치
